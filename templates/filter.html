Filters

  • Filters provide a way to format the data we display to the user.
  • Filters can be used in HTML file by using the '|' pipe or in javascript file by $filter service.
  • To pass an argument to a filter in HTML file, use colon sign ':' after the filter name.

For instance, letâ€™s say we want to capitalize our string. We can either change all the characters in a string to be capitalized, or we can use a filter. 
		<div> {{name | uppercase} </div>
We can also use filters from within JavaScript by using the $filter service. For instance, to use the lowercase JavaScript filter: 
	app.controller('DemoController', ['$scope', '$filter',
		function($scope, $filter) {
			$scope.name = $filter('lowercase')('Ari');
		}
	]);
    There are few in-built filters available:
  • Currency
    • The currency filter formats a number as currency. In other words, 123 as currency looks like: 
    		 <div> {{ 123 | currency } </div>
    Currency gives us the option of displaying a currency symbol or identifier. The default currency option is that of the current locale; however, we can pass in a currency to display
  • Date
    • The date filter allows us to format a date based upon a requested format style. The date formatter provides us several built-in options.
  • Function
    • Runs a function on the data and returns a non falsy output. It will run the function over each element of the array, and the results that return as non-falsy will appear in the new array. For instance, selecting all of the words that have the letter e in them, we could run our filter like so: 
    		 <div>{{ ['Ari', 'Lerner', 'Likes', 'To', 'Eat', 'Pizza'] | filter:'e' } </div>
    Output: ["Lerner","Likes","Eat"]
  • JSON
    • It takes javascript object and returns a string The json filter will take a JSON, or JavaScript object, and turn it into a string. This transformation is very useful for debugging purposes: 
    		 <div>{{ {'name': 'Ari', 'City': 'San Francisco'} | json } </div>

  • Number

  • String
    • It will accept all elements that match against the string. If we want all the elements that do not match the string, we can prepend a ! to the string.
  • Object
    • Compares the properties of the objects. It will compare objects that have a property name that matches, as with the simple substring match if only a string is passed in. If we want to match against all properties, we can use the $ as the key.
You can also specify custom filters by first creating a module and then calling filter() method on the module object.
